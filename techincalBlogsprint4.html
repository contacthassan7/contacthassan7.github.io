Tell your non-tech friend a story about a time you:

Since this course started, I get stuck so many times again and again even did repetitive mistakes. But the real question is how to get unstuck. I’ve had a fair share of moments when I think I’ll nail this task] and I started to write the code somehow stuck in between.

With sprint 4 I can give an example of FizzBuzz challenge. It is quiet a unique challenge in itself.

It requires to write a code which says if a number is a multiple of 3 it says Fizz and if a number is a multiple of 5 it says Buzz and if the number is a multiple of 3 and 5 = 15 it should say FizzBuzz.

Now I tried to pseudo code on paper first and it seems very easy and my approach was straight forward, use if else statement and nail the solution. The solution looks like this

    // if number is divisible by 3 then return fizz
    //else if number is divisible by 5 then return buzz
    //else if number is divisible by 3 && 5 then return fizzbuzz
    //else return the number

So, fizz and buzz were displaying but FizzBuzz was not displaying at all. I fell into the hole of searching and google makes sure when you are puzzled you never get the right answer.

By this time, I tried pseudo code it helps writing down my approach but what to do when I’m stuck. Then I tried rubber duck method, list down the question and took the shower and then tried to explain the question to tissue roll. And I still didn’t get it, seriously. Then I looked into sample solutions and from there I got the idea on changing the position of ‘if’ statement and now whole equation makes sense. And the code is running like a charm.

I learned, sometimes you didn’t get the answer easily. And what appears simple might not be. But changing the environment and going through problem solving techniques equips you with a lot of more knowledge. And you will eventually get there.
